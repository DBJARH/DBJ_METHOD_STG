## “Why We Must Reach Level 3 Architecture Maturity”

Below is a concise, authoritative narrative you can use when presenting this diagram.

## **1. This is the Required Operating Model for Enterprise Architecture**

Point to the diagram and emphasise:

> “This isn’t a suggestion—this is the **standard organisational model defined by The Open Group** for running architecture at a mature level. For us to deliver consistent outcomes, we need to align our organisation to this structure.”

This model defines:

*   **Governance Bodies**
*   **Project/Portfolio Governance**
*   **Roles & Responsibilities**
*   **Architecture Professionals**
*   **Skills, Knowledge & Professional Development**
*   **Enterprise Continuum**
*   **Architecture Repository**

In other words, it describes the “minimum viable architecture organisation” for any serious business.

## **2. We Need to Operate at Least at *Level 3 Maturity***

What Level 3 means in simple terms:

*   Architecture is **repeatable**
*   Processes are **standardised and documented**
*   Roles & responsibilities are **defined and assigned**
*   Work is **governed**, not ad‑hoc
*   There is an **architecture repository**
*   The business gains **predictable outcomes**

This directly connects to the ICL Boot Camp:

> “The Boot Camp exists to establish the Level 3 capabilities—skills, governance, repeatable processes, and roles—that are shown in this diagram.”

## **3. The Diagram Shows the Core Components We Must Build**

Break it down for the team:

### 🔹 **Governance Bodies**

Set direction, priorities, and measure success.  
ICL must implement this to avoid reactive or inconsistent delivery.

### 🔹 **Project/Portfolio Governance**

Where architecture work gets validated and guided.  
Ensures architecture is connected to real business and delivery outcomes.

### 🔹 **Skilled Resource Pool**

Where we define:

*   The roles we need
*   The skills required
*   Professional development paths

Right now, this is missing and must be created.

### 🔹 **Architecture Repository**

A single place to store and reuse:

*   Standards
*   Patterns
*   Models
*   Reusable building blocks

This is essential for scaling.

### 🔹 **Enterprise Continuum**

Provides classification and alignment of architecture assets.  
Without this, everything becomes bespoke and chaotic.

## **4. Our Boot Camp = Establishing These Foundations**

This makes it clear:

> “Step 1 / Boot Camp is not just training. It’s how we *build* the architecture organisation shown in the model so we can reach Level 3 (and beyond) and deliver effectively.”

You can position it as:

| Boot Camp Focus           | Architecture Capability  |
| ------------------------- | ------------------------ |
| Roles & responsibilities  | Skilled resource pool    |
| Standard engagement model | Governance               |
| Templates & artefacts     | Architecture repository  |
| Repeatable processes      | Level 3 maturity         |
| Skill-building            | Professional development |

## **5. The Ask: Everyone Must Engage in Reaching Level 3**

Final message you can deliver:

> “As an organisation, we must operate at a **minimum baseline of Level 3** to ensure quality, repeatability, and trust. The diagram is our target state. The Boot Camp is how we get there. But it only works if everyone participates.”
